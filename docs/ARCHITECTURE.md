# Архитектура KeyFrog

## Назначение

KeyFrog - локальный desktop-менеджер паролей. У приложения нет серверной части: vault создается, хранится и сбрасывается только на компьютере пользователя.

Основные пользовательские сценарии:

- первичная настройка с созданием мастер-пароля;
- вход в приложение по мастер-паролю;
- полный сброс vault-а, если мастер-пароль забыт;
- создание, просмотр, копирование, редактирование и удаление записей;
- генерация надежных паролей с настраиваемой сложностью;
- сохранение паттернов генерации;
- настройка языка, темы, блокировки и очистки буфера обмена.

## Технологический стек

- **Desktop-оболочка:** Tauri 2
- **Прикладная логика:** Rust
- **Frontend:** React, TypeScript, Vite
- **Стилизация:** Tailwind CSS и локальные UI-примитивы
- **Локальное хранилище:** SQLite
- **Доступ к SQLite:** rusqlite
- **KDF:** Argon2id
- **Шифрование секретов:** XChaCha20-Poly1305
- **Генерация случайных значений:** CSPRNG операционной системы

## Модель безопасности

Мастер-пароль не сохраняется. При создании vault-а приложение выполняет следующую последовательность:

1. Генерирует случайную соль для Argon2id.
2. Выводит из мастер-пароля ключ шифрования ключа.
3. Генерирует отдельный ключ шифрования данных.
4. Шифрует ключ данных через XChaCha20-Poly1305.
5. Сохраняет в SQLite только служебные данные: соль, nonce и зашифрованный ключ данных.

При разблокировке из введенного мастер-пароля снова выводится ключ. Затем приложение пытается расшифровать ключ данных. Если расшифровка не проходит, введенный мастер-пароль считается неверным.

Записи vault-а шифруются как JSON-payload целиком:

```json
{
  "serviceName": "Example",
  "login": "user@example.com",
  "password": "secret"
}
```

Метаданные SQLite, например идентификаторы записей и временные метки, остаются видимыми. Если в будущем понадобится шифрование всего файла базы данных, можно рассмотреть переход на SQLCipher или другой механизм whole-file encryption.

## Сброс vault-а

Восстановления мастер-пароля нет. Это намеренное ограничение локального шифрования: если пользователь забыл мастер-пароль, расшифровать существующие данные нельзя.

Сценарий сброса:

1. Пользователь подтверждает удаление vault-а.
2. Приложение удаляет локальный файл хранилища.
3. Состояние разблокированной сессии очищается.
4. При следующем запуске пользователь создает новый vault и новый мастер-пароль.

## Rust-модули

- `crypto` - Argon2id, генерация случайных байтов, AEAD-шифрование и расшифровка.
- `db` - путь к SQLite, создание схемы, работа с метаданными.
- `vault` - жизненный цикл vault-а и операции с записями.
- `generator` - генерация паролей на основе настроек пользователя.
- `models` - типы запросов и ответов для Tauri-команд.
- `error` - преобразование ошибок приложения в строки для frontend.
- `lib` - регистрация Tauri-команд и общее состояние приложения.

## Frontend-модули

- `app` - верхний уровень приложения, маршрутизация между setup, unlock и unlocked-состоянием.
- `features/auth` - экраны создания и ввода мастер-пароля.
- `features/vault` - список записей, карточки записей и форма редактирования.
- `features/generator` - генератор паролей и сохраненные паттерны.
- `features/settings` - локальные настройки, тема, язык, reset vault-а.
- `components/ui` - небольшие переиспользуемые компоненты интерфейса.
- `lib/tauri` - типизированные обертки над Tauri-командами.
- `lib/i18n` - локализация интерфейса.
