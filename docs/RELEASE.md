# Релиз v0.1.0

Этот файл описывает ручную публикацию первого релиза KeyFrog на GitHub.

## Перед релизом

1. Убедиться, что приложение собирается:

```bash
npm run build
cd src-tauri
cargo check
cd ..
npm run tauri -- build
```

2. Проверить, что установщик появился в папке:

```text
src-tauri/target/release/bundle/
```

3. Проверить базовые сценарии:

- создание vault-а;
- вход с верным мастер-паролем;
- отказ входа с неверным мастер-паролем;
- добавление записи;
- копирование логина и пароля;
- показ и скрытие пароля;
- редактирование записи;
- удаление записи;
- генерация пароля;
- сохранение паттерна генерации;
- смена темы и языка;
- сброс vault-а.

## Создание GitHub Release

1. Открыть репозиторий на GitHub.
2. Перейти в раздел **Releases**.
3. Нажать **Draft a new release**.
4. Выбрать или создать тег:

```text
v0.1.0
```

5. В поле **Release title** указать:

```text
KeyFrog v0.1.0
```

6. В описание релиза добавить:

```markdown
Первая рабочая версия KeyFrog - локального менеджера паролей для ПК.

Основное:
- локальное зашифрованное хранилище;
- вход по мастер-паролю;
- сброс vault-а при потере мастер-пароля;
- управление записями;
- генератор паролей;
- сохранение паттернов генерации;
- русская и английская локализация;
- светлая и темная тема.

Перед использованием для важных данных рекомендуется провести дополнительное тестирование и аудит безопасности.
```

7. Прикрепить установщик из `src-tauri/target/release/bundle/`.
8. Нажать **Publish release**.
